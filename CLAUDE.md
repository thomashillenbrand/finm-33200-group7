## What we're building

An agentic system that produces auditable historical truthfulness profiles for public companies. Given an earnings call transcript, the system extracts forward-looking management claims; it then grades whether each claim was realized by checking the same company's subsequent SEC filings (10-Q, 10-K, 8-K).

Three deliverables:
1. A verification agent
2. Per-firm truthfulness profiles for 4 firms — **profile-assembly step still unbuilt** as of 2026-05-25 (see open item #8); the two verdict halves that feed it now exist.
3. A validation gold set — **as built (2026-05-25) this is LLM-labeled** (GPT-5.5 + rubric via `verifier.autolabel`), *not* hand-labeled: a deliberate, time-constrained substitution flagged as a shortcoming / future-work item. The labeler (GPT-5.5 + rubric) is a different model from the graded agent (gpt-5.1, no rubric), which limits but does not remove model-vs-model circularity.

## Scope (locked-in)

- **Sample:** 4 firms × ~20 earnings calls each over 2020–2025 = ~80 calls
- **Firms:** Tesla (auto/EV), Amazon (tech/consumer), Coca-Cola (consumer staples), Eli Lilly (healthcare/pharma)
- **Claim categories:** Numerical guidance (graded against Compustat) + capital allocation: buybacks, dividends, capex plans, debt (graded against subsequent 10-Q balance sheet, cash flow, and 8-Ks)
- **Verification source:** SEC EDGAR exclusively — no news/web APIs
- **Pipeline:** transcript → typed-claim extraction → SEC-filings-based verification → graded verdict (verified / partially verified / contradicted / not yet resolvable) + cited evidence
- **Labeling workflow (load-bearing — don't change):** the agent surfaces candidate evidence *without proposing a verdict*; human labelers read the evidence and assign verdicts independently. Letting the agent's verdict bias the labeler creates circularity in the evaluation.
- **Fallback:** if capital allocation grading hits roadblocks, drop to numerical-guidance-only.

Iteration 3's Compustat-backed numerical grader landed on 2026-05-24 as a separate package: `src/autochecker/`. Two-stage flow per claim — stage 1 (LLM) screens whether the claim is even about a Compustat figure; stage 2 (LLM) compares it against a per-ticker quarterly panel sliced to `call_date < datadate ≤ horizon_end_date` plus four pre-call quarters as a YoY base. YTD cash-flow fields (`capxy`, `prstkcy`, `dltisy/dltry`, `dvy`, `oancfy/ivncfy/fincfy`, `sstky`) are converted to per-quarter deltas before the LLM sees them. Same evidence/verdict mode split as the verifier — `--mode evidence` (default) returns Compustat citations + a neutral comparison paragraph; `--mode verdict` adds a label. Reads `VERIFIER_AGENT_MODEL` from `.env` (no new env var). Uses the raw OpenAI SDK (`client.beta.chat.completions.parse`) rather than LangChain — autochecker has no agent loop or completion cache to integrate with, and the `sec_filings` conda env does not carry langchain. Per-claim citations are scrubbed against the actual sliced panel to drop hallucinated `(datadate, field)` pointers. Validated on a 590-claim run of `data/claims/51_full_run.csv` (290 went to stage 2; 119 verified / 98 contradicted / 49 partial / 23 not-yet-resolvable / 1 insufficient_data). Use `python -m autochecker.run --claims <csv> [--mode evidence|verdict] [--tickers TSLA,KO] [--limit N]`; see README.md "Compustat verification (workstream C, iter-3)".*
